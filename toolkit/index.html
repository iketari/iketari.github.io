Что это?

Кросс-проектные UI компоненты

Зачем?

  1. Кнопочки должны быть одинаковыми на всех проектах
  2. Верстать кнопочку нужно только один раз
  3. Кнопочки необходимо разрабатывать без привязки к проекту
  4. Дизайнеры хотят контролировать внешний вид всех кнопочек

Это понятно, а что внутри?

Пакеты? Что вы в них храните?

toolkit - это 3 репозитория (пакета)

  1. mail/toolkit (jam)
  2. mail/toolkit-bem (npm)
  3. mail/toolkit-common (jam) - разбиваем на отдельные пакеты

Почему jam?

  1. Работает по package.json и вообще похож на npm
  2. Сам строит конфиг для RequireJS
  3. Позволяет устанавливать пакет из git ветки
  4. Не требует серьезного serverside

Проблемы?

  1. Синхронизация
  2. Распространение
  3. Unit-тесты. Их нет.
